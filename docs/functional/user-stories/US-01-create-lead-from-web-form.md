# US-01 Создание лида из веб-формы с валидацией и дедупликацией по телефону и email

## ID
US-01

## Название
Создание лида из веб-формы с валидацией и дедупликацией по телефону и email.

## User story
Как система CRM,  
я хочу принимать заявку из веб-формы, проверять обязательные поля и дедупликацию по телефону и email,  
чтобы корректно регистрировать новый лид без дублей в активной работе.

## Бизнес-ценность
- Сокращение потерь лидов при первичном обращении с сайта.
- Снижение количества дублей и ручной очистки базы.
- Единая и предсказуемая точка входа для новых лидов.

## Actor
Система.

## Trigger
Поступает HTTP-запрос из веб-формы на создание лида.

## Предусловия
- Интеграция веб-формы с CRM API доступна.
- В запросе переданы обязательные поля: `full_name`, `phone`, `email`, `consent=true`.
- Формат `email` должен быть валидным.
- Для дедупликации используются нормализованные значения `phone` и `email`.

## Postconditions
- Если активный дубликат не найден, создан новый лид со статусом `Новый` и `source=web`.
- Если найден активный дубликат (`Новый`, `В работе`, `Отложен`) по `phone` или `email`, новый лид не создается, возвращается признак дубликата и `existing_lead_id`.
- Если совпадения есть только по закрытым лидам (`Ложный`, `Отказ`), создается новый лид с заполнением `reopen_of_lead_id`.
- Если `phone` и `email` указывают на разные активные лиды, возвращается конфликт для ручной проверки менеджером.

## Основной сценарий
1. Веб-форма отправляет данные лида в CRM.
2. Система валидирует обязательные поля и формат `email`.
3. Система нормализует `phone`.
4. Система проверяет наличие активного лида с тем же `phone` или `email`.
5. Если активного дубликата нет, система создает новый лид:
   - `status = Новый`;
   - `source = web`;
   - фиксирует `created_at` и `updated_at`.
6. Система возвращает успешный ответ с `lead_id` и текущим статусом.

## Edge cases / ограничения
- Если отсутствует обязательное поле или некорректный формат данных, запрос отклоняется (`400`).
- Если найден активный дубликат по `phone` или `email`, возвращается `409 duplicate`.
- Если `phone` и `email` совпали с разными активными лидами, возвращается `409 conflict`.
- Если наблюдаются частые повторные отправки за короткий интервал, допускается базовый anti-spam guard (`429`).
- Техническая ошибка на стороне CRM возвращается как `500`.

## Критерии приемки
- При валидном payload и отсутствии активного дубликата создается новый лид со статусом `Новый` и источником `web`.
- При отсутствии `email` или невалидном формате `email` создание лида не выполняется, возвращается `400`.
- При совпадении `phone` или `email` с лидом в статусах `Новый`/`В работе`/`Отложен` новый лид не создается, возвращается `409` и `existing_lead_id`.
- При совпадении `phone` или `email` только с лидом в статусах `Ложный`/`Отказ` создается новый лид с заполненным `reopen_of_lead_id`.
- При конфликте идентичности (`phone` и `email` указывают на разные активные лиды) возвращается `409` с требованием ручной проверки.
- В ответе на успешное создание возвращаются идентификатор лида и его текущий статус.
