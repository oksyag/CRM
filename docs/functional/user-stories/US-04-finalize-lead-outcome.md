# US-04 Финализация лида: отложить, конвертировать в клиента или закрыть

## ID
US-04

## Название
Финализация лида: отложить, конвертировать в клиента или закрыть.

## User story
Как менеджер по продажам,  
я хочу зафиксировать итог обработки лида: отложить контакт, конвертировать лид в клиента или закрыть лид,  
чтобы по каждому лиду был понятный результат и следующий шаг.

## Бизнес-ценность
- По каждому лиду фиксируется однозначный результат обработки.
- Для отложенных лидов сохраняется дата следующего контакта.
- Для закрытых лидов фиксируется причина, что повышает прозрачность работы.

## Actor
Менеджер.

## Trigger
Менеджер завершает очередной этап работы с лидом в карточке и выбирает целевое действие.

## Предусловия
- Лид существует в системе.
- Лид находится в активной обработке (`В работе`) или в статусе `Отложен` для повторного контакта.
- Используются только согласованные статусы: `Новый`, `В работе`, `Отложен`, `Успешно`, `Ложный`, `Отказ`.
- Для закрытия в статусы `Ложный` или `Отказ` менеджер указывает причину (свободный текст).

## Postconditions
- При выборе `Отложен` у лида установлен статус `Отложен` и заполнено `follow_up_at`.
- При выборе `Успешно` лид получает финальный статус `Успешно`, создается связанный `Client`.
- При выборе `Ложный` или `Отказ` лид получает соответствующий финальный статус и заполненную причину закрытия.
- В карточке лида обновляется `updated_at`.

## Основной сценарий
1. Менеджер открывает карточку лида в статусе `В работе` (или `Отложен` при повторном контакте).
2. Менеджер выбирает один из вариантов финализации:
   - отложить контакт (`Отложен`) и указать `follow_up_at`;
   - конвертировать лид (`Успешно`);
   - закрыть лид (`Ложный` или `Отказ`) и указать причину.
3. CRM валидирует выбранный переход и обязательные поля для целевого статуса.
4. CRM сохраняет новое состояние лида.
5. Если выбран статус `Успешно`, CRM создает запись клиента, связанную с лидом.
6. CRM отображает обновленную карточку лида с итоговым статусом.

## Edge cases / ограничения
- Если для статуса `Отложен` не указан `follow_up_at`, операция отклоняется (`400`).
- Если для статусов `Ложный`/`Отказ` не указана причина закрытия, операция отклоняется (`400`).
- Повторная конвертация уже конвертированного лида запрещена (`409`).
- Из финальных статусов (`Успешно`, `Ложный`, `Отказ`) переход обратно в активные статусы запрещен (`409`).
- При внутренней ошибке сохранения возвращается `500`.

## Критерии приемки
- При переводе в `Отложен` обязательно сохраняется `follow_up_at`, иначе операция не выполняется (`400`).
- При переводе в `Успешно` создается связанный клиент и лид фиксируется в финальном статусе.
- При переводе в `Ложный` или `Отказ` без причины операция не выполняется (`400`).
- После установки финального статуса (`Успешно`, `Ложный`, `Отказ`) повторный перевод лида в `В работе` не допускается (`409`).
- После успешной финализации менеджер видит в карточке лида итоговый статус и актуальные данные.
