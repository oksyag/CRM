# Глоссарий CRM для обработки лидов

Термины и обозначения, используемые в документации проекта.

| Термин | Определение |
|--------|-------------|
| **CRM** | Система для приёма, учёта и обработки лидов менеджером. |
| **Лид (`Lead`)** | Запись входящего или вручную заведённого контакта, проходит согласованный жизненный цикл и статусы до итога (конвертация или закрытие). |
| **Клиент (`Client`)** | Запись клиента после успешной конвертации лида из статуса `Успешно`; связь с лидами — по правилам модели данных. |
| **Менеджер** | Основная роль пользователя в сценариях обработки; также справочная сущность **`Manager`** (активность — `is_active`). |
| **Система** | Роль при автоматической регистрации лида из веб-формы (валидация, дедупликация, сохранение). |
| **`status` / статус лида** | Обязательное поле жизненного цикла. Допустимые значения: `Новый`, `В работе`, `Отложен`, `Успешно`, `Ложный`, `Отказ`. В DDL PostgreSQL — тип `lead_status`. |
| **Активные статусы** (при создании лида и дедупликации) | Подмножество статусов, при которых новый лид с тем же `phone` или `email` блокируется: `Новый`, `В работе`, `Отложен`. |
| **Финальные статусы** | `Успешно`, `Ложный`, `Отказ` — дальнейший перевод лида назад в `В работе` или `Отложен` запрещён согласованными правилами. |
| **`source` / источник лида** | Канал поступления. Допустимые значения: `web`, `phone`, `chat`, `social`, `manual`. В DDL PostgreSQL — тип `lead_source`. При ручном создании недопустим `web`; `web` — для интеграции веб-формы. |
| **`assignee_id` / ответственный менеджер** | Ссылка на менеджера, закреплённого за лидом. |
| **`full_name`** | Обязательное ФИО или имя контакта на лидах и клиенте. |
| **`phone`** | Обязательный телефон; перед дедупликацией нормализуется правилами CRM. |
| **`email`** | Обязательный email; формат проверяется приложением; для дедупликации нормализуется (регистр, trim по правилам CRM). |
| **`follow_up_at`** | Дата и время следующего контакта; обязательна при статусе `Отложен`. |
| **`close_reason`** | Свободный текст причины закрытия; обязателен при статусах `Ложный` и `Отказ`. |
| **`comment`** | Необязательный текстовый комментарий в карточке лида (в т.ч. пометки по дубликату или источнику). |
| **`reopen_of_lead_id`** | Ссылка на предыдущий лид при повторном обращении после закрытия в `Ложный` или `Отказ`. |
| **`consent`** | Признак согласия на обработку данных; обязателен при создании лида из веб-формы (`consent=true`); для ручного создания не используется. |
| **Дедупликация** | Поиск совпадения при создании лида по нормализованному `phone` **или** `email` среди уже существующих лидов с учётом правил активных/закрытых статусов. |
| **`existing_lead_id`** | Идентификатор уже существующего лида при срабатывании правила активного дубликата. |
| **Конфликт идентичности** | Когда один и тот же запрос сопоставляется так, что по `phone` находится один **активный** лид, а по `email` — **другой активный** лид; новый лид не создаётся до ручного разрешения. |
| **Нормализация** (`phone`, `email`) | Приведение значений к единому виду перед проверкой дубликатов. |
| **Интеграция веб-формы** | Передача данных заявки с сайта в CRM по API (HTTPS), канал создания лида со `source=web`. |
| **Карточка лида** | Экран/представление с полными данными одного лида. |
| **Список лидов** | Перечень лидов с минимальной фильтрацией по `status`, менеджеру и источнику (`source`). |
| **Конвертация лида** | Успешное завершение обработки: статус `Успешно` и наличие связанного клиента **`Client`.** |
| **Веб-посетитель** | Внешний actor: пользователь сайта, отправляющий заявку через публичную веб-форму. Учётной записи в CRM не имеет. |
| **`is_active`** | Поле сущности `Manager`. Признак доступности менеджера для назначения на лид. При `false` менеджер не отображается в выборе ответственного. |
| **`lead_id`** | Поле сущности `Client`; ссылка на исходный лид. Уникально (`UNIQUE`): один клиент создаётся ровно для одного конвертированного лида. |
| **`request_id`** | Идентификатор HTTP-запроса, фиксируемый в логах CRM для отладки и трассировки ошибок без полных персональных данных. |
| **`Idempotency-Key`** | HTTP-заголовок, поддерживаемый POST-эндпоинтами CRM (`/web/leads`, `/leads`, `/leads/{id}/convert`). При повторе запроса с тем же ключом CRM возвращает результат предыдущей обработки и не создаёт новый ресурс. |
